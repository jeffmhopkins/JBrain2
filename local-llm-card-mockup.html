The meter's denominator is the model's window — change it above and this tracks it, because both read the same persisted value (no drift).

Lifecycle: Stage → Load → Unload. Each model owns a color on the memory bar; within its segment the solid block is weights and the lighter block is context (KV cache) at the chosen window. While idle, set the context window (the configured -c), then Stage the model — its segment appears hatched (proposed, not yet resident) so you can see whether the set fits the 128 GB host before committing; the context block grows/shrinks live as you change the window. Load makes it resident (hatch removed) and locks the window for the running process; Unload frees it and reopens editing.
