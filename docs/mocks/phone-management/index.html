Location → Phones — management mockups

Three interactive variants for managing the family's paired phones. Open each on a phone-width window; tap around — pairing, re-pair/roll code, rename, revoke, delete all respond.

What's being fixed
  • Phones only — the Add device (OwnTracks) path is dropped.
  • Roll the token / rotate the key on an already-paired phone — unified into one re-pair action that mints a fresh QR code; the phone redeems it and its key rotates automatically (a paired phone can only take credentials by redeeming a code).
  • Rename a phone and delete it (distinct from revoke), so the revoked clutter from the current screen has a home.
  • Organised around family members — whose phone is whose.
A — Family roster + device hub grouped · sheet

Phones grouped under each family member; tapping a phone opens a full management sheet (status + key state, re-pair, rename, danger zone). Revoked phones fold under a per-person disclosure. Most "family-first".
B — Swipe-action rail filter · swipe

Flat list with an Active / Revoked filter. Swipe a phone left for a 4-action rail (re-pair · rename · revoke · delete) — reuses the settled home-note / chats swipe paradigm. Rename inline, destructive actions arm a tap-again confirm, undo toast. Densest.
C — Inline accordion expand · credential

Each phone unfolds in place to a panel led by a credential-lifecycle strip (key active / aging / waiting / disabled), then an action grid; the roll-code QR opens right inside the card. Puts the key's state front-and-centre.
