Settings · LLM

Mock F — capacity-gauge hero + docked resident-group clusters. A radial gauge of the one ~128 GB unified pool is the centerpiece: a solid green arc for resident weights+KV, an amber arc for staged (warm), a violet slice for image VRAM, and the rest free headroom. Models live as tiles docked into RESIDENT / STAGED / IDLE clusters, each labeled with a count and GB subtotal. Tapping a tile focuses it — the gauge grows a dashed preview wedge of what loading it would cost and a verdict line answers "fits" or honest "over by X GB"; acting (stage/load/unstage/unload) re-docks the tile into its new cluster and the gauge re-settles. Decision-support first: it answers "will it fit / what do I unload" before it's a list. Catalog (install/uninstall + queue) lives in a bottom sheet; image VRAM rides the same gauge as a violet slice.

On-box memory
RESIDENT
Nothing loaded. Load a staged model to make it resident.
STAGED · warm
Nothing staged. Staging keeps weights warm so the next request skips the cold load.
IDLE · on device
Everything installed is in memory.
Catalog

Every catalog model. Install pulls the weights; Uninstall removes them and frees the disk. Queued installs apply on your next update.
