On-box model states

Four states, one lifecycle. Catalogue and Available are tabs; Resident is a tab; Staged is a transient preview — tap Stage on any available model to see whether loading it would evict what's already resident.

The four states

Catalogue

Every model JBrain knows about — installed (weights on disk) or uninstalled. Install pulls the weights; uninstall reclaims the disk.

Available

Installed models the router is allowed to swap in and out. Only these compete for memory. Add/remove from the roster without touching disk.

Resident

Loaded in unified memory right now. The subset the box is actually holding.

Staged

Not a stored state — a preview. Staging an available model runs the eviction check and shows what would leave to make room, before you commit the load. Cancel and nothing changed.

Strix Halo · unified memory

Box: 128 GB unified memory · keep-free floor 25% → 96 GB usable ceiling (the dashed marker). Staging evicts the fewest, biggest-footprint models first until the new one fits under the ceiling — the same rule the residency coordinator runs on the server (ensure_room).
